# Template Registry

## Purpose
Reusable blueprints and stubs that can be copied and adapted for new workflows. Each template includes parameterized fields marked with `{{PLACEHOLDER}}`.

## Templates

| Name | Type | Description |
|------|------|-------------|
| `git-commit-message.txt` | Git | Standard commit message template |
| `n8n-error-handler.json` | n8n | Reusable error-handling sub-workflow pattern |
| `workflow-doc-template.md` | Docs | Template for documenting a new workflow |

## How to Use
1. Copy the template file to your working location
2. Replace all `{{PLACEHOLDER}}` values with your specific details
3. Adapt the structure as needed
4. If you make significant improvements, update the template

## How to Add a Template
1. Create the template file with `{{PLACEHOLDER}}` for variable fields
2. Add it to the table above
3. Ensure it's generic enough to be reused
